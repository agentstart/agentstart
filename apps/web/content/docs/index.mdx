AgentStart is a TypeScript AI agent framework. It ships with a full set of tools and database adapters out of the box, making it easy to build production-ready agents with persistent memory, tool execution, and type-safe client integration.

## Why AgentStart?

Building AI agents often requires piecing together multiple libraries and writing boilerplate code for persistence, tool management, and client-server communication. AgentStart solves this by providing:

<Cards>
  <Card title="Unified Agent Runtime" description="Built on top of AI SDK with oRPC for type-safe communication" />
  <Card title="Database Adapters" description="First-class support for Drizzle, Prisma, Kysely, and MongoDB" />
  <Card title="Built-in Tools" description="File operations, shell commands, and task management out of the box" />
  <Card title="Built-in Components" description="Pre-built conversation and layout primitives via shadcn-style registry" />
  <Card title="Type-Safe Client" description="Type-safe client integration with automatic state management" />
  <Card title="Framework Integration" description="Ready-to-use adapters for Next.js, Node.js, and more" />
</Cards>

## Quick Example

Here's a minimal example of an AgentStart agent:

```ts
// Server
import { agentStart } from "agentstart";
import { Agent, agentTools } from "agentstart/agent";
import { drizzleAdapter } from "agentstart/memory";
import { createOpenRouter } from "@openrouter/ai-sdk-provider";

const openrouter = createOpenRouter({
  apiKey: process.env.MODEL_PROVIDER_API_KEY!,
});

const agent = new Agent({
  model: openrouter("x-ai/grok-4-fast"),
  instructions: "You are a helpful assistant.",
  tools: { ...agentTools },
});

export const start = agentStart({
  agent,
  memory: drizzleAdapter(db, { provider: "postgresql", schema }),
  appName: "my-app",
});
```

```tsx
// Client
import { useThread } from "@/lib/agent-client";

export function Chat() {
  const { threadId } = useParams();
  useThread(threadId);
  
  return <ChatInterface />;
}
```
